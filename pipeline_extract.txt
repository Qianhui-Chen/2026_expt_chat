前后端分离
建立数据库，数据库表头包括用户的登录名、点击行为、每个界面停留时间、聊天记录、时间戳
步骤一：
Figma: anger_登录页->后端获取用户ID 
用户ID由A/B+数字构成，A对应anger条件下的提示词和界面设计要素，anger组的AI前端设计需要在AI部分的输出框加上4秒的动画，浮现出💢表情图，在4秒的时间里💢和聊天框本身需要抖动几下，以呈现出愤怒的视觉效果。💢不要挡住文字，影响正常的沟通
；B对应neutral 条件下的提示词和界面设计要素，non-anger组的AI前端设计聊天框只需要自然浮现即可；奇数对应position和用户一致的提示词；偶数对应无position的提示词。
Anger x aligned position
Non-anger x aligned position
Anger x ambiguous position
Non-anger x ambiguous position 
用户ID正则表达式对应
A / B + 奇数 → aligned
A / B + 偶数 → ambiguous
A → anger，B → neutral
提示词（完整文档见 docs/PROMPTS.md；代码在 backend/app/conditions.py）
- 第 1–5 轮：RESPONSE_REQUIREMENTS_EARLY + 情绪/复述 early + 立场 early + GUIDANCE_EARLY
- 第 6 轮：RESPONSE_REQUIREMENTS_FINAL + 情绪/复述 final + 立场 final（强化 position 对建议的约束）+ FINAL_ROUND_ADVICE
- aligned 立场：early 认可用户；final 再次站用户一侧，且 3 条建议不得削弱用户正当性
- ambiguous 立场：early 中立引导换位思考；final 再次中立，且 3 条建议须体现多元视角、不偏用户
前端设计要素
愤怒组的AI前端设计需要在AI部分的输出框加上4秒的动画，浮现出💢表情图，在4秒的时间里💢和聊天框本身需要抖动几下，以呈现出愤怒的视觉效果。💢不要挡住文字，影响正常的沟通
B对应neutral 条件下的提示词和界面设计要素，不愤怒组的AI前端设计聊天框只需要自然浮现即可
